# Stitch Screen Instructions

## Explore

- Redesign result and comparison areas to avoid compressed single-line summaries.
- Give the main text block higher width priority than badges and trailing actions.
- Allow scenario summaries to wrap to two lines.
- If a row becomes crowded, stack the action below instead of shrinking the text.
- Keep the value readable at all times.
- Use icon-only Share and More controls in the utility area; keep visible text for primary actions.

## Library

- Make each saved scenario row a text-first vertical stack.
- Keep the summary readable in two lines.
- Keep the timestamp secondary.
- Put actions below the summary if width is limited.

## Share Card

- Use a fixed-canvas-safe layout that does not depend on one-line metadata.
- Keep the current value as the hero.
- Break scenario metadata into separate blocks.
- Make comparison items readable as individual rows, not fused strings.
- Keep disclaimer and snapshot note intact in the footer.

## Trust Notes

- Present each trust statement as a separate line.
- Split freshness and provider information if a single sentence becomes too long.
- Protect financial meaning over compactness.

## Loading Overlay

- Make the title and body wrap cleanly.
- Keep the card centered and balanced under longer strings.

## Copy Guidance

- shorten labels before changing typography
- avoid explanatory buttons
- one support sentence should explain one thing only
- do not weaken finance disclaimers to make them fit
